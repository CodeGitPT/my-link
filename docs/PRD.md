# 프로젝트 요구사항 정의서 (PRD) - 마이링크 (MyLink)

## 1. 프로젝트 개요

*   **프로젝트명:** 마이링크 (MyLink)
*   **목적:** 사용자가 운영하는 다양한 웹사이트, 블로그, 소셜 등의 링크를 하나의 페이지로 모아 간편하게 공유할 수 있는 극도로 심플한 프로필 링크 서비스.
*   **대상 사용자:** 다양한 채널의 팔로워를 한 곳으로 모으거나 자신만의 온라인 활동들을 하나의 접속 링크로 깔끔하게 안내하려는 개발자 및 크리에이터 등 모든 사용자

## 2. 기술/디자인 컨셉

*   **UI/UX 디자인:** shadcn/ui 기반의 모던하고 일관성 있는 미니멀 컴포넌트 디자인 (테마 변경 없이 단일 고정 디자인 제공)
*   **인증 및 DB 설계:** Firebase Authentication (구글 소셜 로그인 전용) 및 Firebase Firestore 활용

## 3. 핵심 기능 목록 (MVP)

1.  **회원 관리:** Firebase 구글 소셜 로그인(One-Click), 계정 정보 관리 및 로그아웃
2.  **텍스트 프로필 및 식별자 (인라인 편집):** 
    * **고유 슬러그 (DisplayName):** 접속 주소(URL)로 사용되는 식별자. 구글 로그인 시 사용된 지메일(Gmail) 아이디의 앞부분을 추출하여 기본값 설정 (`mylink.com/displayname`)
    * **표시 이름 (Username):** 방문자에게 직관적으로 보여지는 실제 사용자 이름 (예: 홍길동, Jane Doe 등)
    * 텍스트 형태의 짧은 소개글 등록 (이미지 업로드 기능 원천 배제)
    * **인라인 편집 지원:** 화면 내에서 텍스트를 직접 클릭해 즉각적으로 수정 가능
3.  **링크 관리 및 자동 파비콘 (인라인 편집):** 
    * 새로운 대상 링크 추가 및 표출
    * **인라인 편집:** 등록된 링크의 제목(Title)을 클릭해 바로 수정 가능
    * **파비콘 기능:** 외부 요소를 최소화하고, **구글 파비콘 API**를 활용하여 대상 사이트의 아이콘을 자동으로 불러와 버튼에 표시
    * 원터치 링크 삭제 기능 포함

*(참고: 추후 업데이트를 통해 개별 링크의 '클릭 조회수' 리포팅 기능이 추가될 예정입니다.)*

## 4. 세부 기능 및 데이터베이스(DB) 제원

### 4.1 로그인 및 회원 관리
*   별도의 절차 없이 시스템 상호작용은 오직 **Firebase 구글 로그인**으로만 수행됩니다.

### 4.2 프로필 설정 (명명 규칙 및 URL 연동)
*   **DisplayName (URL 슬러그 역할):** 구글 소셜 로그인 시 연동된 이메일 계정(예: `dev123@gmail.com`)의 앞부분 아이디(`dev123`)를 파싱하여 이 값을 URL 슬러그로 **자동으로 최초 설정**합니다. 이 값이 본인의 마이링크 고유 주소(`mylink.com/dev123`)가 됩니다.
*   **Username (화면 노출 이름):** URL 규칙에 구애받지 않는 자유로운 텍스트로, 한글 띄어쓰기 등 일반적인 텍스트가 모두 허용되며 프로필 상단에 실제 이름으로 표출됩니다.
*   인라인 편집을 통해 두 값 모두 변경 가능하며, `displayName` 변경 시 URL 접속 주소도 함께 변경되므로 중복 검사가 필수적으로 진행되어야 합니다.
*   Firestore DB에서는 사용자 문서 안에 `displayName`, `username`, `bio`(소개글) 필드가 명확하게 분리되어 저장됩니다.

### 4.3 링크 관리 (서브 컬렉션)
*   **데이터 모델 구조:** 각 사용자가 등록하는 개별 링크 항목 데이터들은 모두 해당 사용자 문서 하위의 **서브 컬렉션(Sub-collection)** 단위로 분리 저장됩니다.
*   링크 순서 변경(드래그 앤 드롭)이나 임시 비활성화(토글) 같은 부가 기능은 지원하지 않습니다.

### 4.4 디자인 (shadcn/ui 환경)
*   사용자 프로필 페이지는 이미지 요소 없이 텍스트(표시 이름/소개글)와 파비콘이 포함된 링크 컴포넌트만으로 렌더링됩니다.
*   **테마 변경 제외:** 다크/라이트 모드 전환이나 컬러 시스템 커스터마이징 기능을 모두 없애고 프로젝트에 내장된 단일 테마로만 동작합니다.
*   모든 필터, 미리보기, 테마 설정 패널을 걷어내고 **"클릭 후 즉시 수정(인라인 편집)"**이라는 통일된 UX에 초점을 맞춥니다.
